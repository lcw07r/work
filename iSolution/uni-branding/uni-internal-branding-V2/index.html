The Template

An internal branding template for University of Southampton applications based on a responsive twelve column grid.

The Grid

The base grid is a response twelve column grid that is available in both fixed and fluid widths. Columns scale in discreet steps retaining their positioning until the screen size is that of the average mobile browser (767px or below) at which point the columns will stack.

To start your grid first add a .container element, this will set the overall width of the grid, before adding a .row element to start your first row of columns and then add to this any number of .column or .columns elements up to a maximum combination of twelve.

By default there is no spacing between rows however you can add the class .small, .medium and .large to the row to add 11px, 22px or 44px of padding to the top and bottom of the row.

Nested Columns

Columns can also be nested within existing columns, however how these behave depends on whether you are using a fixed or fluid template. In the fluid template each column can be further sub-divided in to twelve columns, however the fixed template only allows a column to be sub-divided in to as many columns as the column itself. Nesting columns is simple, simply add a new .row element to the column you wish to contain the nested columns and add your columns to the new row.

Offset Columns

If you need to add more column spacing to your grid you can add the class .offset-by-xxx to a column, doing so will add the specified number of column(s) width of padding to the column. You must take column offsets in to account when constructing your grid as the total columns including offsets cannot exceed twelve.

Source Ordering

Using the classes .push-by-xxx and .pull-by-xxx your columns can be displayed in a different order to that defined in the code, .push-by-xxx will push the column by the specified number of columns to the right and .pull-by-xxx will pull the column to the left n.b. when viewed on a mobile browser the columns will stack in their source order.

Media Queries

In conjunction with the base grid there are a number of media queries that allow you to target various devices, the media queries are targetted at maximum and minimum widths rather than devices or orientations

  • Desktop and Tablet Landscape: (min-width: 960px)
  • Tablet Portrait: (min-width: 768px and max-width: 959px)
  • All mobile devices: (max-width: 767px)
  • Mobile landscape: (min-width: 480px and max-width: 767px)
  • Mobile Portrait: (max-width: 479px)

There are also three helper classes that can be used within your designs to hide certain elements from being displayed at the various different screen sizes:

  • .hide-on-desktop hides the element from being displayed on all browsers with a screen resolution greater than 960px
  • .hide-on-tablet hides the element from being displayed on all browsers with a screen resolution less than 960px and greater than 768px
  • .hide-on-mobile hides the element from being displayed on all browsers with a screen resolution less than 768px

Typography

Typography is based around a line-height of 22px and font size of 13px, in addition to this the size, padding and margin of all standard elements are based upon divisions of this line height to ensure that your content has a natural rhythm. If images or site specific elements are sized around this division then they will not break the flow of the site.

Tables

By default there are no formatting styles associated with the table tag this is to ensure that no styling is improperly associated with any tables created by third party plugins such as calendars. To add styling to your table add the class .table to your table and styling for captions, headers and columns as well as applying zebra stripig to alternate rows.

Forms

By default form inputs are designed to utilise the full width of their parent element with labels being positioned above their related input. To display an inline form with the label either to the left or right of the input apply add the class .left or .right to the label and set a width on the relevant input.

Content Boxes

There are four multipurpose boxes that can be added to your code to highlight content, simply create a div element with the class .box and optionally one of the following classes .information, .confirmation, .warning. Content boxes can also be further enhanced by adding the class .rounded which will add rounded corners to the element, and you can add a title to the content box by adding an element with the class .title which will add a colour co-ordinated title bar.

Any ordered or unordered list can be converted in to a pager by simply adding the class .paging to the parent list, the current page is highlighted by adding the class .current to the list element which also prevents it from being clicked.

Menus

Any unordered list can be converted in to a vertical or horizontal menu by placing it in a container with the classes .menu horizontal or .menu vertical. Levels of navigation are achieved by nesting further ordered or unordered lists within the parent element. By default the horizontal menu only supports two levels of navigation whereas the vertical menu supports three.

Images

There are three classes that you can add to an image to change the way that they are displayed and can be combined in any combination.

Scaled

Adding the class .scale ensures that the image will scale with its parent container. 

<img src="..." class="scale"/>

Bordered

Adding the class .bordered adds a thin border and box shadow. 

<img src="..." class="scale bordered"/>

Rounded

Adding the class .rounded adds rounded corners to the image. 

<img src="..." class="scale rounded"/>
